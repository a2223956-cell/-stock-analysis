# 股票综合分析框架 V2.0

> 融合两套系统精华：
> - Hermes量价分析：深度K线量价关系解读、5源新闻情绪、主力行为逐K线追踪
> - daily_stock_analysis：可插拔策略框架、7条核心理念、评分体系、大盘复盘、风控机制

---

## 一、7条核心交易理念（不可违反）

1. **严进策略** — 不追高，乖离率(MA5偏离)<5%才考虑入场
2. **趋势交易** — MA5>MA10>MA20多头排列，顺势而为
3. **效率优先** — 关注筹码结构好、资金配合的票
4. **买点偏好** — 优先在MA5/MA10附近回踩买入，不追突破
5. **风险排查** — 高风险利空一票否决，宁错过不做错
6. **量价配合** — 成交量必须验证价格运动，无量上涨=虚假
7. **龙头放宽** — 真正的板块龙头可适当放宽标准，但必须有辨识度

## 二、分析流程（8步法）

**⚠️ 执行前置要求: 接到分析任务后，第一步必须告知用户将调用哪些skill，再开始执行。例如: "计划调用的skill: 1.stock-analysis 2.a-stock-data 3.mx-moni"。用户明确要求每次做事前先看到skill列表。**
**简化规则**: 同一会话中首次分析必须完整列出skill列表；后续分析(同一会话内)若skill无变化，可简写为"继续调用stock-analysis + a-stock-data + mx-moni"或省略（已在上下文中）。见2026-07-14实测。

### 批量分析工作流（2+只票）

当用户一次问多只股票("深度分析A和B")时：

1. **告知skill列表** → 告诉用户将调用哪些skill
2. **并行委托** → 用 delegate_task 的 tasks 数组，每只票一个子agent，互不阻塞
3. **等待结果** → 子agent完成后自动回传
4. **读取报告** → 用 read_file 读取每只票的 .md 报告
5. **更新watchlist** → 每只票都更新（已在watchlist的更新，不在的新增）
6. **呈现给用户** → 逐票展示 + 末尾横向对比表

⚠️ 不要串行委托（一只完成再委托下一只），并行总耗时约等于单只票。

### "还拿不拿"/"该卖吗"/"加仓"类问题：成本价前置 + 结论先行

当用户问"还拿不拿"、"该卖吗"、"要不要跑"、"是否可以加仓"时：

**⚠️ 第一步(不可跳过): 确认成本价**
在委托子agent或开始分析之前，必须先问用户成本价。
不能先做分析再回头问——浪费一轮交互且分析方向可能需要基于成本价调整。
用clarify工具一次性问，给几个成本区间选项(接近现价/小幅浮亏/中度浮亏/深度套牢)。

**第二步: 结论先行**
```
== 🔴/🟡/🟢 结论: [减仓/持有/加仓]! [一句话操作] ==
```

然后按B4持仓成本分析模板输出。

### 步骤-1: 历史分析复盘（深度分析时必做）

当用户说"深度分析XX"时，在正式分析前先检查历史记录：

**操作**:
1. 搜索历史报告:
   ```bash
   search_files(pattern="*分析报告*", target="files", path="/home/harry/stock-reports/{股票名}_{代码}")
   ```
   + `session_search(query="{股票名} {代码} 分析")`

2. 如果找到历史报告:
   - 读取报告中的操作建议/支撑位/阻力位/主力判断/综合评分
   - 拉取当前数据(腾讯行情)对比价格变化
   - 输出复盘:
   ```
   == 历史复盘: {股票名}({代码}) ==
   上次分析: {日期} | 当时价格: {XX.XX}
   当前价格: {XX.XX} | 区间涨跌: {+X.XX%}

   上次结论验证:
     ✅/❌ {结论1} → 实际: {验证结果}
     ✅/❌ {结论2} → 实际: {验证结果}
     ...

   上次遗漏:
     {哪些因素没考虑到，导致判断偏差}

   本次需重点关注:
     {基于上次遗漏+当前新情况的特别关注点}
   ```

3. 如果没有历史报告:
   - 跳过，正常执行后续步骤

**⚠️ 关键规则**:
- 复盘结果放在分析报告最前面(在市场环境之前)
- 复盘必须诚实: 上次判断对了就说对了，错了就说错了
- 遗漏分析是核心价值: 哪些因素没考虑到？为什么没考虑到？下次如何避免？
- 如果上次的支撑位/阻力位被击穿，本次分析必须重新计算
- **⚠️ 历史报告的K线数据也可能错误**: 子agent可能编造K线数据(如通富微电79.39案例: 子agent将6/25高点79.38错误归因为7/10开盘价79.39,41处引用全部错误)。读取历史报告时需对关键K线数据做基本校验: 开盘价应在最高最低之间,涨停价=前收盘×涨跌幅限制。详细案例见 `references/kline-data-fabrication-case-study.md`

### 步骤1: 前复权处理（必做，在所有分析之前）

⚠️ 2026-06-15教训: 不复权会导致支撑阻力位、均线偏离、量价关系全部失真。

**操作**:
1. 先查除权记录: `client.xdxr(symbol=code)` 获取近期分红/转增
2. 确认分析周期内是否有除权(特别是近60日内)
3. 如果有除权(category=1):
   - 前复权: 旧行情(除权前) = 原始价 / 复权因子 - 每股派息
   - 复权因子 = 1 + songzhuangu/10 (送转股)
   - 除权后行情无需调整
4. 所有均线、支撑阻力、偏离度计算都基于复权价
5. mootdx的adjust='qfq'有pandas兼容bug，需手动计算

```python
# 示例: 5/21 10转2 + 每10股派0.75元
ex_date = pd.Timestamp('2026-05-21')
factor = 1.2  # 1 + 2/10
dividend_per_share = 0.075  # 0.75/10

mask = bars.index < ex_date
bars.loc[mask, 'adj_close'] = bars.loc[mask, 'close'] / factor - dividend_per_share
bars.loc[mask, 'adj_high'] = bars.loc[mask, 'high'] / factor - dividend_per_share
bars.loc[mask, 'adj_low'] = bars.loc[mask, 'low'] / factor - dividend_per_share
```

如无除权则原始价=复权价，可跳过此步。

| 平(<1%) | 平 | 强 | 🟡 可能有独立催化，关注 |

- 大盘强+板块弱 → 板块轮动可能不在这个方向，谨慎
- 大盘弱+板块强 → 独立行情，但需警惕补跌

### 步骤3: 量价关系分析（核心，第一优先）

**⚠️ 单股深度分析(session)时，必须用本skill的K线规则手动检查近3-5日组合形态，不能只依赖scan_candidate.py的输出。scan_candidate.py V4.2已支持3日组合形态检测(双针探底/十字星确认/看涨吞没等)，但单股分析需要更精细的手动判断(如验证形态的严格条件、检查信号是否已被后续K线确认/失效)。**

**目标**: 通过逐日K线+成交量判断主力处于什么阶段

**主力行为4阶段识别框架**:

| 阶段 | 量价特征 | 典型K线 |
|------|---------|--------|
| 建仓期 | 缩量阴跌后放量阳线；底部温和放量价不跌 | 底部长下影阳线 |
| 洗盘期 | 缩量回调（量缩价不跌）；地量出现=洗盘尾声 | 缩量十字星 |
| 拉升期 | 温和放量上涨（量价齐升）；放量突破平台 | 放量中阳线 |
| 出货期 | 天量天价；放量下跌；冲高回落上影线长 | 高位大阴线 |

**⚠️ 每个判断必须附带具体K线实例**:
- 日期 + 开/收/高/低
- 成交量 + 与前日对比(放大/缩小百分比)
- 为什么这根K线说明主力在做X

**量价背离（趋势拐点预警）**:
- 顶背离：价格创新高但成交量未创新高 → 见顶信号
- 底背离：价格创新低但成交量未创新低 → 见底信号

**量价评分**: 1-5分
- 5分: 底部放量建仓+量价同步上涨
- 4分: 缩量企稳+温和放量反弹
- 3分: 量价中性，无明确信号
- 2分: 有出货迹象但未完全确认
- 1分: 天量天价/放量暴跌/量价全面恶化

### 步骤4: 资金流向验证

**目标**: 用资金流数字验证量价判断是否准确

- 主力资金连续净流入 → 印证建仓/拉升判断
- 主力资金连续净流出 → 印证出货判断
- 量价说建仓但资金在流出 → 判断可能有误，需重新审视

**关键规则**: 资金流是辅助验证，量价才是主力真实意图

### 步骤5: 技术面+均线系统

**均线多空排列判断**:
- 多头排列: MA5>MA10>MA20>MA30 → 趋势向上
- 空头排列: MA5<MA10<MA20<MA30 → 趋势向下
- 纠缠态: 均线交叉缠绕 → 方向不明

**关键技术指标**:
- MACD: 金叉/死叉、零轴上下
- RSI: 超买(>70)/超卖(<30)
- 量比: >2放量 / <0.5缩量
- 换手率: >10%异常活跃 / <1%冷门

### 步骤6: 估值+基本面

**估值检查**:
- PE(TTM) vs 行业均值
- PB vs 每股净资产
- PEG = PE / 净利润增速(%) → <1便宜 / 1-1.5合理 / >1.5贵
- 前向PE = 当前价 / 机构一致预期EPS

**基本面关注**:
- 净利润增速（同比）
- ROE水平
- 机构覆盖度（有无近期研报）
- 机构一致预期EPS

### 步骤7: 概念板块完整映射（必做，不可跳过）

**目标**: 查清个股的完整概念归属，为后续对比分析提供基础

**操作**:
1. 获取个股全部概念标签(同花顺reason / 东财概念板块eastmoney_concept_blocks())
2. 特别标注近期市场热门概念是否在列表中
3. 区分"核心概念"(有真实业务) vs "蹭概念"(无实质业务)
4. 记录概念纯度评级(★★★~★★★★★)

**⚠️ 常见错误**: 只取最显眼的一个概念就下结论
→ 协鑫能科案例: 遗漏"算电协同"核心战略，只标"抽水蓄能"，
导致对比分析结论严重偏差。必须获取完整概念列表。

### 步骤8: 消息面+风险筛查

**消息面情绪打分**:
- 利好/利空条数统计
- 关键新闻重要性评估
- 消息面总情绪: 偏多/中性/偏空 + 得分

**风险筛查（RiskAgent模式）**:
必须检查的风险项:
- [ ] 大股东减持/质押
- [ ] 业绩预亏/变脸
- [ ] 监管处罚/立案调查
- [ ] 行业政策利空
- [ ] 限售解禁（30天内）
- [ ] 估值异常（PE>100或为负、PB>10）
- [ ] 技术破位（死叉、跌破关键支撑）

**风险分级**:
- high: 重大风险（造假/诉讼/大规模减持）→ 一票否决
- medium: 显著风险（业绩不及预期/解禁/板块利空）→ 降级信号
- low: 轻微风险（分析师下调/小额减持）→ 记录但不阻断

## 三-B、新仓买入评估模板（用户问"现在是否值得买入"）

当用户问"XX现在能不能买/值得买入吗"时，使用此模板。
**与加仓决策(第四节)的区别**：加仓是已持仓用户问要不要加仓，新仓是从零建仓。

### 买入信号三要素（缺一不可）

李成刚策略的买入条件是**三个条件同时满足**，不是"满足一个就可以"：

| 条件 | 要求 | 不满足时 |
|------|------|---------|
| ① 缩量比 | < 0.8 | 抛压未减轻，不是洗盘尾声 |
| ② K线信号 | 十字星/长下影/双针探底 | 缺乏止跌确认信号 |
| ③ R:R比 | > 1.0（最好>2.0） | 风险远大于收益，不值得冒险 |

**⚠️ R:R比是最常被忽略但最关键的条件：**
- R:R = (阻力位-现价) / (现价-支撑位)
- R:R < 1 = 下跌空间 > 上涨空间 = 不划算
- 即使缩量+十字星都满足，R:R < 1 也不应买入
- 常见陷阱：MA20偏离过大(>15%)的票，R:R几乎必然<1

### R:R比计算与解读

```python
# R:R = 上涨空间 / 下跌空间
resistance = 前高 or 阻力位
support = MA20 or 前低 or 支撑位
rr = (resistance - current_price) / (current_price - support)

# 解读
# R:R > 2.0 → 优秀（上涨空间是下跌的2倍以上）
# R:R 1.0-2.0 → 可接受
# R:R < 1.0 → 不值得冒险（下跌空间大于上涨空间）
# R:R < 0.5 → 极差（几乎必亏的赔率）
```

**⚠️ R:R边界情况: 价格跌破或贴近支撑位时**

当current_price <= support时,分母为零或负数,R:R公式失效。
这发生在:价格已跌破MA20/前低等支撑位,或正在测试支撑位。

正确处理:
```python
if current_price <= support:
    # 价格已破支撑或贴着支撑,R:R公式分母<=0
    # 用当日最低价或下一级支撑位(MA30/前期低点)重新计算
    new_support = day_low or MA30 or prior_low
    if current_price > new_support:
        rr = (resistance - current_price) / (current_price - new_support)
    else:
        rr = 0  # 全面破位,无法计算
```

案例(凯盛科技7/6): 现价25.10 ≈ MA20(25.06),用当日低点24.31做支撑 → R:R=2.08
但如果跌破24.31且无新支撑确认 → R:R无效,应观望

**为什么高位票R:R必然差：**
- 价格在MA20上方很远 → support(支撑位)距离远 → 下跌空间大
- 前高距离近 → resistance(阻力位)距离近 → 上涨空间小
- 结果：R:R极低，即使方向判断正确也赚不到钱

### 评估流程

```
Step 1: 扫描脚本获取技术指标
  → scan_candidate.py 输出: 缩量比/MA20偏离/K线形态/R:R/评分

Step 2: 三要素检查（按优先级）
  ① R:R > 1.0? → 不满足直接PASS，不看其他条件
  ② 缩量比 < 0.8? → 不满足需等待
  ③ 十字星/长下影? → 不满足需等待

Step 3: 如果三要素都满足，再看：
  - 均线排列（多头加分，空头减分）
  - 资金流向（净流入加分，净流出减分）
  - 消息面（利好加分，利空减分）
  - 估值（PE合理加分，泡沫减分）

Step 4: 综合判断
  - 三要素全满足 + 均线多头 + 资金流入 → 🟢 可建仓
  - 三要素全满足 + 其他条件有瑕疵 → 🟡 小仓位试探
  - 三要素任一不满足 → 🔴 等待，不买入
```

### 输出模板

```

  R:R比: X.XX ✅/❌ (阈值1.0)

三要素检查:
  ① 缩量比<0.8: ✅/❌
  ② K线信号: ✅/❌
  ③ R:R>1.0: ✅/❌
  三要素满足: X/3

R:R详细计算:
  现价: XX.XX
  阻力位(前高): XX.XX → 上涨空间 +X.X%
  支撑位(MA20/前低): XX.XX → 下跌空间 -X.X%

### 关键原则
- **R:R是第一道门槛**：不满足直接PASS，不看其他条件
- **宁可错过，不要追高**：缩量到位但R:R差 = 等回调

2. **板块近期表现**：
   - 板块指数涨跌幅（vs大盘）

   - 板块资金流向（主力在流入还是流出）

### 市场全景扫描工作流（用户问"扫描市场"时使用）

当用户问"扫描一下市场情况"时，按以下流程执行：

**Step 1: 大盘指数** — mootdx quotes获取上证/深证/创业板实时数据

**Step 2: 涨跌家数统计** — 核心步骤，用于判断赚钱效应
```python
from mootdx.quotes import Quotes
import pandas as pd
client = tdx_client()  # 见 a-stock-data V3.3.0 Prerequisites

# 获取股票列表（注意: stocks()只返回code/name/pre_close元数据，不含价格）
sh = client.stocks(market=1)  # 沪市
sz = client.stocks(market=0)  # 深市

# 过滤A股
sh_a = sh[sh['code'].str.match(r'^[06]')]
sz_a = sz[sz['code'].str.match(r'^[03]')]
all_codes = pd.concat([sh_a, sz_a])['code'].tolist()

# 批量获取实时报价（分批100只，mootdx quotes支持批量）
all_data = []
for i in range(0, len(all_codes), 100):
    batch = all_codes[i:i+100]
    try:
        df = client.quotes(symbol=batch)
        if df is not None and not df.empty:
            all_data.append(df)
    except:
        pass

# 计算涨跌统计
result = pd.concat(all_data, ignore_index=True)
result = result[result['price'] > 0]
result['chg'] = (result['price']/result['last_close']-1)*100

total = len(result)
up = (result['chg'] > 0).sum()
down = (result['chg'] < 0).sum()
limit_up = (result['chg'] >= 9.9).sum()
limit_down = (result['chg'] <= -9.9).sum()
med_chg = result['chg'].median()

# 涨幅分布
bins = [(-100,-7),(-7,-5),(-5,-3),(-3,0),(0,3),(3,5),(5,7),(7,10),(10,100)]
labels = ['<-7%','-7~-5%','-5~-3%','-3~0%','0~3%','3~5%','5~7%','7~10%','>10%']
for (lo,hi), label in zip(bins, labels):
    count = len(result[(result['chg']>=lo)&(result['chg']<hi)])
```

⚠️ 已知限制:
- mootdx `stocks()` 返回的DataFrame只有 `code/volunit/decimal_point/name/pre_close` 五列，**不含实时价格**
- 必须用 `quotes()` 批量获取实际价格，分批100只避免超时
- 6000+只股票全量获取约需30-60秒
- 部分股票(price=0)需要过滤

**Step 3: 板块/概念热点** — 用mx-search或web_search获取当日市场主线

**Step 4: 关注个股快扫** — mootdx quotes批量获取用户持仓/关注个股

**Step 5: 资金面** — 两融余额、融资买入占比等

**关键判断指标**:
- 中位数涨幅 > 0 → 赚钱效应好
- 中位数涨幅 < -1% → 亏钱效应明显
- 涨停/跌停比 → 情绪温度
- 上涨比例 > 60% → 普涨; < 30% → 结构性分化
- 成交额 → 量能活跃度

### ⭐ 板块横向对比分析工作流 (同板块多股分化原因分析)

当用户问"为什么同板块A涨B不涨"/"板块内走势差异"时，使用此6维度分析框架:

#### Step 1: 概念重叠+催化差异
- 列出每只票的完整概念标签(东财eastmoney_concept_blocks / 同花顺reason)
- 标注重叠概念 vs 独有概念
- 独有催化: 谁有近期新增催化(研报/政策/订单/国企改革等)
- 概念纯度: 谁是龙头(营收>50%) vs 蹭概念(<5%)

#### Step 2: 资金流向对比(核心)
- 逐日对比主力净流入/流出(东财push2his 20日)
- 识别"板块内资金切换": 同一天A净流入+B净流出 = 资金从B切到A
- 计算N日累计净流入, 排序谁是资金宠儿
- **关键指标**: 6日主力累计净流入差异可解释>60%的走势分化

#### Step 3: 估值压力差异
- PE/PB横向对比, 谁贵谁便宜
- 高PE(>150)是最大负担 — 市场对高估值更谨慎
- 定增/增发增加股本摊薄压力
- 亏损股炒"困境反转" vs 高PE股炒"成长预期"

#### Step 4: 技术面正反馈/负反馈
- 多头排列(越涨越吸引跟风盘) vs 空头排列(越跌越引发止损盘)
- 均线排列对比: 谁多头/谁空头/谁纠缠
- 价格在25日区间中的位置(高位80%+ vs 低位20%-)

#### Step 5: 主力行为阶段差异
- 谁在建仓/拉升(放量+资金流入) vs 谁在出货(高位+资金流出)
- 见顶信号: 长上影线+天量+后续资金持续流出
- 主力建仓信号: 巨量资金单日涌入(>总市值3%)

#### Step 6: 股价/市值/概念属性差异
- 低价股(10-20元)更受游资偏好(想象空间大)
- 国企改革催化 vs 民营企业(缺乏额外催化)
- 市值大小影响资金撬动难度

  N日涨跌      x%        x%        x%
  PE_TTM       xx        xx        xx
  主力N日流    x亿       x亿       x亿
  均线排列     多/空/缠  多/空/缠  多/空/缠
  25日位置     x%        x%        x%
  ──────────────────────────────────────────

走势分化原因(按重要性排序):
  ① [核心原因]: 资金选择差异/催化差异/估值差异
  ② [次要原因]: 技术面/主力行为/概念纯度
  ③ [辅助原因]: 股价/市值/属性

投资启示:
  板块内选谁: 票A > 票B > 票C (理由)
  操作差异: A可持有 vs B应减仓 (理由)
```

**⚠️ 常见陷阱**:
- 不能说"板块退潮"来解释个股弱于板块 → 必须区分板块整体 vs 个股问题
- 同板块不同命运的核心是"资金选择了谁", 不是基本面差异
- 资金流向数据(6日累计)是最硬的证据, 其他维度是辅助解释

### 概念炒作股 vs 真实业务股 判断框架

⚠️ 2026-06-12 教训: 沃格光电/彩虹股份/五方光电都是"玻璃基板概念"，
但只有沃格和戈碧迦有真实半导体TGV业务，彩虹做的是LCD面板玻璃

判断标准:
```
1. 公司自己怎么描述这块业务?
   → 搜索"F10公司概况" + 最新异常波动/风险提示公告
   → 如果公司说"尚处于早期阶段/研发验证/未形成规模" = 画饼

2. 该业务占营收比例多少?
   → 搜索"公司年报/半年报 分业务收入"
   → 营收占比<5% = 概念大于实质

3. 技术路线是否真的对口?
   → 搜索"公司产品 技术路线 vs 概念要求"
   → 彩虹做LCD面板玻璃 vs 半导体TGV封装玻璃 = 完全不同赛道

### 出货完成度评估（量化判断底部）

当量价+资金流已确认出货后，需要量化评估出货是否已接近尾声，判断是否可以开始关注底部信号。

**计算方法**:
```
流入阶段: 拉升/涨停期间主力累计净流入(如三次涨停日合计+31.5亿)
流出阶段: 出货期间主力累计净流出(如6/17-7/8合计-34.74亿)
净额 = 流出阶段 - 流入阶段

判断:
  净额 < 流入额 → 出货进行中(还有筹码要出)
  净额 ≈ 流入额 → 出货基本完成(筹码已大部分换手)
  净额 > 流入额 → 出货超额完成(可能有新增抛压,如融资盘/止损盘)
```

**出货完成后的底部信号检查**:
1. 抛压收窄: 最近1-3日流出金额大幅缩小(如从-2亿收窄至-0.3亿)
2. 缩量企稳: 成交量萎缩至拉升期的30-50%
3. 长下影探底: 盘中暴跌后尾盘拉回(下方有承接)
4. 板块/大盘强但个股弱: 资金偏好不在该方向,需要等轮动

⚠️ **出货完成 ≠ 可以买入**。出货完成后可能进入:
- 底部横盘(2-3周缩量整理) → 最佳情况
- 继续阴跌(散户止损盘+融资盘) → 还需等待
- V型反弹(超跌反弹) → 短线机会但趋势未反转

**案例(中国西电 2026-07-09)**:
- 流入: 6/15+6/16+6/22三次涨停日主力合计+31.5亿
- 流出: 6/17-7/8主力合计-34.74亿
- 净额(-34.74) > 流入(+31.5) → 出货超额完成
- 但7/6-7/8仍有-3.54亿流出 → 尾部抛压未完全消除
- 7/8流出-0.33亿(大幅收窄) → 抛压接近尾声信号
- 7/9盘中暴跌至12.61后拉回13.31 → 长下影探底(但需次日确认)

## 九、技术形态速查

### 双针探底 (Double Needle Bottom)

两根K线均带长下影线，低点接近(差距<2%)，出现在下跌趋势末端。

5项检查条件:
1. 两日都有长下影(下影/实体 > 1.5x) ← 严格要求两天都是
2. 下影线低点接近(差距<2%)
3. 实体较小(十字星或小阴小阳)
4. 第二日收盘≥第一日
5. 出现在下跌趋势中(价格低于前5日均价)

⚠️ 常见误判: 只有一根长下影+一根普通小阴线，不构成双针探底。
只有单根长下影的叫"锤子线"(Hammer)，也是止跌信号但弱于双针。

### 组合形态检测 (近3日K线，单股分析必做)

⚠️ scan_candidate.py已实现以下7种组合形态的自动检测，但单股深度分析时仍需手动验证。

**1. 双针探底** (+1.0分)
  条件: 连续2日都有长下影(锤子线), 低点差距<2%
  含义: 下方承接极强, 双重探底确认

**2. 十字星确认** (+0.8分)
  条件: 前日十字星 + 今日阳线且收盘>前日收盘
  含义: 多空平衡后多头占优, 止跌确认

**3. 锤子线确认** (+0.8分)
  条件: 前日锤子线 + 今日阳线且收盘>前日收盘
  含义: 下探后承接+确认反弹

**4. 看涨吞没** (+0.7分)
  条件: 昨日阴线 + 今日阳线, 今日实体完全包裹昨日实体
  (今日close > 昨日open 且 今日open < 昨日close)
  含义: 多头一举吞没空头, 反转信号

**5. 双针探底(变体)** (+0.7分)
  条件: 前日锤子 + 今日锤子, 低点差距<3%
  含义: 类似标准双针但其中一天形态不标准

**6. 连阴后十字星** (+0.6分)
  条件: 连续2日阴线后出现十字星
  含义: 空头力竭, 潜在止跌(需次日确认)

**7. 反包长上影** (+0.6分)
  条件: 昨日阴线且上影线>实体×40% + 今日阳线收复昨日开盘价
  含义: 昨日冲高回落被今日消化, 反弹信号

**信号失效判定:**
  - 昨日有信号(十字星/锤子) + 今日阴线且跌破昨日最低价 → 信号失效
  - 2日前有信号 + 连续2日阴线 + 今日跌破2日前最低价 → 信号失效
  - 信号失效后不计入评分, 报告中应明确标注

### 锤子线/倒锤子线单日判定

单根K线：下影线长度 > 实体 × 2，出现在下跌趋势末端。
是止跌信号，但需要后续K线确认(次日不破低点+放量阳线)。

### 其他常见形态(速查)

| 形态 | K线特征 | 含义 |
|------|---------|------|
| 十字星 | 实体<振幅×10% | 多空平衡，方向待定 |
| 长上影阳线 | 上影>实体×2 | 上方抛压重，可能见顶 |
| 放量长阴 | 大阴线+量>前日×1.5 | 主力出货/恐慌抛售 |
| 缩量十字星 | 量<前日×0.5+十字 | 洗盘尾声(配合支撑位) |
| 缩量锤子线 | 量=近30日最低+下影>实体×2+收盘≈最高 | ⭐ 经典底部形态(需次日确认) |

## 九、K线形态识别工具包

### 双针探底（Double Needle Bottom）5项条件检查

适用场景：下跌趋势末端，两日K线低点接近，判断是否出现止跌信号。

```python
# 伪代码 — 每个条件独立判定
def check_double_needle(day1, day2):
    """
    day1/day2 各含: open, high, low, close
    返回: (通过数, 详细判定)
    """
    body1 = abs(day1.close - day1.open)
    body2 = abs(day2.close - day2.open)
    lower1 = min(day1.close, day1.open) - day1.low
    lower2 = min(day2.close, day2.open) - day2.low

    c1 = lower1 > body1 * 1.5 and lower2 > body2 * 1.5  # 两日都有长下影
    low_avg = (day1.low + day2.low) / 2
    c2 = abs(day1.low - day2.low) / low_avg < 0.02       # 低点差距<2%
    c3 = body1 / (day1.high - day1.low) < 0.3 or \
         body2 / (day2.high - day2.low) < 0.3             # 至少一天实体小
    c4 = day2.close >= day1.close                          # 第二日不破前日收盘
    c5 = day1.close < 前5日均价 * 0.97                     # 出现在下跌趋势中
    return sum([c1, c2, c3, c4, c5])
```

判定标准：
- 5/5 → ✅ 标准双针探底
- 4/5 → 🟡 近似双针（检查哪项不满足，降级评估）
- ≤3/5 → ❌ 不构成双针

**⚠️ 常见误判：**
- 6/11只是小阴线(下影0.18 < 实体0.20)不算"针" → 不能和6/12锤子线凑成双针
- 真正的针要求下影线 > 实体 × 1.5
- 低点下移（新低）比低点持平弱，因为说明支撑在往下找

### 锤子线/倒锤子线单日判定

锤子线（底部止跌信号）：
- 条件：下影 > 实体 × 2，上影 < 实体，出现在下跌趋势中
- 含义：盘中大幅下探但尾盘拉回，下方有买盘承接

倒锤子线（顶部信号）：
- 条件：上影 > 实体 × 2，下影 < 实体，出现在上涨趋势中
- 含义：冲高回落，上方抛压重

### MA5突破有效性评估

用户常问"站上MA5能不能加仓"。评估要点：

| 条件 | 权重 | 说明 |
|------|------|------|
| 突破幅度 | 关键 | >1%才有效，<0.5%属于"贴着走"随时回落 |
| 放量确认 | 关键 | 成交量 > 前日 × 1.5 或 > 5日均量 × 1.3 |
| 连续站稳 | 重要 | 至少连续2日收盘 > MA5 |
| 资金配合 | 辅助 | 当日主力净流入 > 0 |
| 均线趋势 | 辅助 | MA10方向(向上=有效突破，向下=假突破概率高) |

**⚠️ 常见陷阱：** 单日微幅站上MA5(+0.29%) + 无放量 + 均线空头 = 不是买入信号，只是技术性反弹。

## 十三、Pitfalls

### 绝不编造数据点(核心原则)
### 绝不编造数据点
复盘/分析时，所有支撑位、压力位、目标价必须来自实际K线数据(mootdx bars)。禁止推测。
用户会追问"这个数据哪里来的"——如果答不上来，说明数据是编造的。

### K线数组索引必须映射日期(2026-07-20教训)

⚠️ 分析彤程新材时,代码中 `klines[7]` 对应的是7/09(第8根K线),但注释写成了"7/08"。
原因: klines[0]=6/30, klines[6]=7/08(跳过周末), klines[7]=7/09。索引到日期的映射不是简单的+1。

正确做法:
```python
# 在打印前先验证日期
for i, k in enumerate(klines):
    print(f"klines[{i}] = {k[0]}")  # 确认索引→日期映射
# 然后再用索引访问
```
⚠️ 特别注意周末/节假日会导致K线日期不连续,索引偏移。

### R:R在自由落体趋势中是陷阱(2026-07-20教训)

⚠️ 彤程新材7/20: R:R=7.29(阻力80-现价66.81)/(66.81-支撑65)=7.29,看起来非常优秀。
但这是陷阱:
1. 支撑位65是当日盘中低点,尚未确认为有效支撑
2. 股价处于自由落体(从102跌至66),支撑位不断被击穿
3. R:R高是因为支撑位"近"(65),但"近"恰恰说明支撑薄弱
4. 如果65破位,下一支撑在60甚至50.78,R:R立刻恶化

正确判断:
```python
# R:R有效性检查
if trend == 'freefall':  # 连续创新低+均线空头
    if support == recent_intraday_low:  # 支撑是今日低点
        print("⚠️ R:R无效: 自由落体趋势中,盘中低点不是有效支撑")
        print("需等待: 连续3日不创新低+止跌K线确认支撑")
# R:R只有在支撑位被验证后才有效(至少1次触及不破)
```

### 锤子线必须配合资金流入才有效(2026-07-20教训)

⚠️ 彤程新材7/14出现"锤子线+极端缩量",7/14报告判断为"底部初步信号"。
但之后7天又跌了-15.3%,信号完全失效。

根因: 锤子线出现时资金仍在持续流出(7/14主力-1.75亿),说明:
- 锤子线只是出货途中的短暂喘息,不是真正的底部
- 极端缩量只能说明"抛压暂时减轻",不代表"买盘进场"
- 没有资金流入配合的锤子线=假信号

正确判断:
```
有效底部信号 = 锤子线/十字星 + 缩量 + 资金连续3日净流入
无效底部信号 = 锤子线/十字星 + 缩量 + 资金仍在流出
→ 后者只是"出货中途的喘息",不是底部
```

案例对比:
  彤程新材7/14: 锤子线+缩量+资金流出 → 假信号,之后跌-15.3%
  正确底部: 锤子线+缩量+资金连续流入 → 真信号

### 两波出货+中继平台模式识别 (2026-07-20双环传动教训)

⚠️ 2026-07-20 教训: 双环传动7/14报告判断"底部整理初期"(评分25.0),但7/17暴跌-9.07%
否定了全部结论。根因: 7/9-7/16的8日整理不是底部,而是出货中继平台。

**两波出货模式**:
```
第一波出货: 天量暴跌(如7/6-7/8, -14.3%)
中继整理:   5-15日窄幅震荡,看似企稳(如7/9-7/16)
第二波出货: 再次暴跌,创新低(如7/17-7/20, -11.2%)
```

**中继平台 vs 真底部 区分**:

| 特征 | 中继平台(假底部) | 真底部 |
|------|----------------|--------|
| 整理时间 | 5-15日 | 15-30日 |
| 整理幅度 | 窄幅(5-8%) | 宽幅(先跌后涨>10%) |
| 资金流向 | 进出交替(有进有出,断断续续) | 连续5日+净流入 |
| 量能 | 缩量后不再放量 | 缩量后温和放量 |
| 均线 | MA5与MA10纠缠,未金叉 | MA5上穿MA10 |
| 创新低 | 整理期间仍创新低(如7/16最低40.23) | 整理期间不创新低 |
| 板块对比 | 弱于板块(板块涨,个股跌) | 同步或强于板块 |

**关键判断规则**:
- 整理期间出现任何一日创新低(收盘或盘中) → 中继平台概率>70%
- 资金"进出交替"(今日+1亿,明日-0.8亿) ≠ 底部吸筹 → 只是热钱进出
- 整理后暴跌>5% → 确认为中继平台,需等下一个支撑

**双环传动案例**:
- 7/9-7/16: 8日整理,看似企稳(7/10反弹+2.4%,资金+1.59亿)
- 但7/16盘中最低40.23(已破7/9低点40.83!) → 中继平台信号
- 7/17暴跌-9.07% → 确认中继平台
- 教训: 7/16的40.23新低是关键预警信号,7/14报告未充分重视

**评分影响**: 当整理期出现创新低时,应将"量价关系"评分从2-3分降至1.5分,
不应出现"底部整理初期"的乐观判断。

### 区分"板块退潮"与"个股出货"（核心分析陷阱）
⚠️ 2026-06-22 教训: 分析协鑫能科时，我说"算力概念整体退潮"导致暴跌，

硬件赛道(CPO/光模块)当天仍在大涨(6/15大面积涨停)。

正确做法：
1. 先确认个股所在细分赛道的具体走势(不能笼统说"XX概念退潮")
2. 区分"板块整体调整" vs "个股独立出货"
3. 如果板块仍在活跃但个股暴跌 → 问题在个股层面(出货/利空/质押)
4. 如果板块和个股同步下跌 → 才是板块级别调整
5. 搜索同板块其他标的验证：如果同板块其他票也在跌=板块问题，如果
   只有这只票跌=个股问题

反面案例: "算力概念退潮"→ 实际CPO/光模块当天涨停，协鑫能科是个股出货。

### scan_candidate.py K线信号只检查当日,需手动回看(2026-07-03发现)

scan_candidate.py的K线形态检测(十字星/长下影) **只检查当日(最后一根)K线**,不回看最近2-3天。

**问题**: 前1-2天出现过有效K线信号(如十字星),但当日是普通阴/阳线时,脚本报"❌无K线信号"。
**案例**: 泛微网络7/1出现标准十字星(body_ratio=5.4%),但7/3是普通阴线(body_ratio=71.8%),脚本报❌。
**正确做法**: scan后手动回看最近3天K线形态:
1. 计算每天的body_ratio(实体/振幅)和lower_ratio(下影/振幅)
2. body_ratio<10% = 十字星, lower_ratio>body*2且lower>upper = 长下影
3. 如果最近2-3天有有效信号且未被后续K线击穿(未破低点) → 仍视为有效
4. 如果后续K线破了十字星低点 → 信号失效

**K线形态速查**:
```
十字星: 实体/振幅 < 10%
长下影: 下影线 > 实体×2 且 下影线 > 上影线
双针探底: 两日都有长下影 + 低点差距<2%
锤子线: 下影 > 实体×2, 出现在下跌趋势末端
```

### 量价分析必须有K线实例
每个主力行为判断必须附带具体哪天的K线+量价数据。
不能只说"主力在出货"而不说"5/27放量长上影阴线，成交量放大14%..."

### 资金流数字只是验证
量价关系是判断主力行为的第一依据，资金流数字是辅助验证。
两者矛盾时优先信量价。

### 加仓决策不能只看估值
PE便宜不等于可以加仓。必须同时满足量价+技术+资金+估值。
"估值合理但主力在出货"= 不能加仓。

### 消息面利好发布时机 / "利好兑现出货"识别框架
注意利好新闻是否配合股价高位发布——可能是出货掩护。
5/27深科技"利好"发布恰在最高点47.03，就是典型案例。

⚠️ 系统性识别框架(2026-07-20通富微电V2报告教训):

**"利好兑现出货"= 利好发布的当天或次日就是股价最高点**

典型模式:
  ① 利好发布(业绩预增/中标/政策) → 股价当天暴涨(涨停或大阳线)
  ② 次日或第2天: 成交量创近期新高(天量) + 价格冲高后回落(天量天价!)
  ③ 之后连续下跌, 利好被完全price in后, 没有新增催化支撑价格

识别信号(缺一不可):
  A. 利好发布后1-2天内出现30日天量 + 30日天价 → 经典见顶
  B. 利好发布后资金并未持续流入(仅当天流入,次日即转出)
  C. 股价已从底部上涨>30%(说明利好已被部分定价)
  D. 同板块其他标的未同步上涨(说明是个股层面出货,不是板块催化)

通富微电案例(2026-07):
  7/14: H1业绩预增288-337%发布 → 收盘77.87(全天最高)
  7/15: 天量293万手+价格84.70(30日最高) = 天量天价见顶!
  7/17: 收盘68.36,资金单日流出-15.40亿
  7/20: 跌至68.60, 从高点84.70跌-19.2%
  → V2报告将"利好兑现"误判为"持续催化",评分80.5分→实际暴跌

### 技术形态筛选：涨停出货信号
"涨停1~3板 + 次日高开低走 + 高倍量阴线"是一组经典的主力出货形态：

完整逻辑链:
1) 涨停板 = 最强广告效应，吸引打板客/散户/游资跟风
2) 次日高开 = 利用隔夜FOMO情绪，给主力更好的出货价格
3) 高开低走收阴 = 主力反手卖出，卖压远大于买压
4) 高倍量(≥2倍) = 大量筹码在高位完成交接(主力→散户)

信号含义: "涨停是诱饵，次日高开低走高倍量是出货确认"
实操意义:
  已持有 → 明天开盘就跑
  观望 → 绝对不要抄底
  量化角度 → 可作为反向筛选/融券做空条件

⚠ 变体: 涨停日资金流反转（不一定高开低走收阴）
2026-06-26 深科技案例: 涨停次日仍收涨+5.83%(不是阴线),
但东财push2his资金流显示主力净流出-13.00亿(涨停日+9.06亿)。
即K线形态看似强势,但资金数据已暴露主力出逃。

识别方法: 对比涨停日与次日的主力资金流向
  涨停日主力流入X亿 + 次日主力流出Y亿 (Y>X) → 确认出货
  即使次日K线收阳,资金流反转也是出货信号
  ⚠ 需要东财push2his资金流数据验证,单看K线会误判

脚本: scripts/zt_scan.py — 全市场扫描(em_zt_pool()+东财K线)
用法: python3 scripts/zt_scan.py
注意: 需unset代理环境变量(unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY)

### 区分"抛压减轻"与"底部吸筹"(2026-06-26 实战)

暴跌后股价横盘不等于底部吸筹,必须用资金流数据区分:

  抛压减轻 (selling exhaustion):
    - 量能大幅萎缩(如缩至暴跌期的70%以下)
    - 价格止跌横盘,低点逐步微升
    - 但资金仍在净流出(只是金额变小)
    - 含义: 卖的人卖完了,没人砸了,但也没人买
    - 操作: 持有等待,不割肉也不加仓

  底部吸筹 (accumulation):
    - 量能萎缩后温和放量
    - 主力资金连续多日净流入(关键区别!)
    - 低点明显抬高,阳线量>阴线量
    - 含义: 主力在主动买入建仓
    - 操作: 可考虑分批建仓

  判断方法: 看资金流方向
  - 整理期资金仍净流出 → 抛压减轻,非吸筹
  - 整理期资金连续3日+净流入 → 吸筹信号
  - 断断续续流入但总体流出 → 热钱进出,非建仓

  ⚠ 协鑫能科案例(2026-06):
  暴跌期(6/1-6/12)资金流出-19.77亿
  整理期(6/15+)资金仍流出-5.81亿
  结论: 不是吸筹,是抛压减轻

### ⚠️ 不能用个股暴跌推断"板块逻辑崩塌"(2026-06-22 教训)

- 协鑫能科暴跌是: ①翻倍后获利回吐 ②个股主力出货 ③大股东质押
- 不是"算力概念崩塌"，是个股层面的行为

**正确分析框架**:
1. 先查板块指数走势(vs大盘) → 判断板块整体强弱
2. 再查个股vs板块的相对强弱 → 判断是个股问题还是板块问题
3. 如果板块强但个股弱 → 个股层面原因(出货/利空/资金)
4. 如果板块弱但个股强 → 个股有独立催化
5. 如果两者都弱 → 可能是系统性风险

**绝对不能说**: "XX概念退潮"/"XX板块崩塌"，除非有板块指数数据支撑。
应该说: "个股在XX概念中回调幅度大于板块，可能是个股层面原因"。

   - 前复权: 旧行情(除权前) = 原始价 / 复权因子 - 每股派息
   - 复权因子 = 1 + songzhuangu/10 (送转股)
   - 除权后行情无需调整
4. 所有均线、支撑阻力、偏离度计算都基于复权价

mootdx的adjust='qfq'参数有pandas兼容性bug(fillna method参数变更)，
需手动计算复权:
```python
# 示例: 5/21 10转2 + 每10股派0.75元
ex_date = pd.Timestamp('2026-05-21')
factor = 1.2  # 1 + 2/10
dividend_per_share = 0.075  # 0.75/10

mask = bars.index < ex_date
bars.loc[mask, 'adj_close'] = bars.loc[mask, 'close'] / factor - dividend_per_share
bars.loc[mask, 'adj_high'] = bars.loc[mask, 'high'] / factor - dividend_per_share
bars.loc[mask, 'adj_low'] = bars.loc[mask, 'low'] / factor - dividend_per_share
```

⚠️ 常见错误:
- 原始价49.99(5/11) vs 原始价49.29(今日) → 看似"逼近前高"
- 前复权41.58(5/11) vs 原始价49.29(今日) → 实际已突破前高18.5%
- 不复权会导致支撑阻力位、均线偏离、量价关系全部失真

### 同板块PE分化是走势分化的最强预测因子(2026-07-20特高压案例)

⚠️ 许继电气(PE=20,+1.87%) vs 中国西电(PE=46,+2.58%) 同属特高压板块,

**核心发现**: 同板块内,PE差异可解释>60%的走势分化:
- 低PE标的(行业均值下方): 下跌动力弱,反弹快,资金愿意进场
- 高PE标的(行业均值上方): 下跌空间大,反弹弱,资金持续出逃

**量化验证**:
  许继PE=20(行业均值25-35下方) → 15日资金流出仅-1.35亿
  西电PE=46(行业均值25-35上方85%) → 15日资金流出-16.43亿(12倍!)
  → 资金自动选择便宜的,抛弃贵的

**操作规则**: 同板块横向对比时,PE差异必须作为第一维度分析:
1. 先比较PE vs 行业均值(谁便宜谁贵)
2. 再比较资金流向(验证PE差异是否导致资金选择差异)
3. 最后看技术面/催化差异(次要因素)

### 评分趋势震荡vs单调下降的含义不同(2026-07-20教训)

⚠️ 通富微电4次评分: 70→38.5→80.5→33.5(剧烈震荡)
   深科技10次评分: 2.20→0.68(单调下降)

**评分趋势的两种模式**:

  震荡模式(通富微电70→38.5→80.5→33.5):
    含义: 分析框架对该股不稳定,可能是:
    ① 股价波动大(涨停/暴跌交替),每次分析时点不同结论不同
    ② 利好/利空交替出现,导致评分反复
    ③ 分析师对"出货vs洗盘"判断困难
    → 震荡=不确定性高,操作难度大,应降低仓位或回避

  单调下降模式(深科技2.20→0.68):
    含义: 趋势明确向下,每次分析都在恶化
    → 单调下降=趋势确认,应坚决执行止损/不抄底

**操作规则**:
  评分震荡→ 🟡 降低仓位,缩小止损区间,等待方向明确
  评分单调下降 → 🔴 坚决执行上次建议(止损/不买)
  评分单调上升 → 🟢 趋势确认,可持有/加仓

### V型反转 vs 拉高出货 的区分方法(2026-06-30教训)

⚠️ 四方达(300179)案例: 6/30低开至52.28(-3.11%)后反弹至59.15(+9.62%)，
上次分析判断"出货期"，但今日V型反转+长下影(6.87元)+短上影(0.25元)，
说明下方承接力极强，不是拉高出货形态。

区分要点:
  V型反转(洗盘后反弹):
    ✓ 下影线远大于上影线(下方承接强)
    ✓ 早盘杀跌后持续走高(不是冲高回落)
    ✓ 缩量或温和放量(非天量)
    ✓ 均线支撑有效(未跌破关键均线)

  拉高出货:
    ✗ 上影线远大于下影线(高位抛压重)
    ✗ 冲高后持续回落(不是V型)
    ✗ 天量成交(换手率>15%)
    ✗ 主力资金净流出(资金流验证)

核心判断指标: 上影线/下影线比值
  下影/上影 > 3 → V型反转概率大(承接强)
  上影/下影 > 3 → 拉高出货概率大(抛压重)
  比值接近1 → 需要结合量能和资金流判断

### ⚠️ 单股分析不能只依赖scan_candidate.py的K线输出 (2026-07-03教训)

scan_candidate.py只检查当日K线形态(is_doji/has_long_lower)，会遗漏：
- 近2-3日的组合形态(双针探底/十字星确认/看涨吞没等)
- 信号失效判定(十字星后被阴线击穿)
- 待确认状态(前日信号尚未确认)

**正确做法:**
1. scan_candidate.py结果作为辅助参考(批量扫描用)
2. 单股深度分析时，必须拉取近3-5日K线数据
3. 按stock-analysis的组合形态规则逐日检查
4. 检查信号是否已被确认或失效
5. 在报告中明确标注组合形态名称和状态

**反面案例:** 泛微网络7/1有标准十字星(body_ratio=5.4%)，但scan只看7/3当天报"无K线"，导致遗漏重要信号。

### 板块内个股分化必须用6大原因框架分析（2026-07-09教训）

⚠️ 四方达案例: 用户问"黄河旋风涨幅完全不同"，本质是板块内个股分化。
不能只分析个股本身，必须横向对比同板块其他标的。

正确做法:
1. 识别同板块核心标的（培育钻石→四方达/黄河旋风/力量钻石）
2. 批量获取行情+K线+资金流，做对比表
3. 按6大原因框架逐一分析差异根源:
   - 催化剂差异（国企改革/政策利好/业绩催化）
   - 资金选择差异（主力在买入谁？低价股更易被游资攻击）
   - 估值压力差异（PE高低/定增摊薄/亏损反而成优势）
   - 技术面差异（多头排列vs空头排列=正反馈vs负反馈）
   - 主力行为差异（建仓拉升vs出货）
   - 概念纯度差异（真龙头vs蹭概念）
4. 结论要可操作："同板块不同命运，核心是资金选择了谁"

反面案例: 只说"A比B强"而不说"为什么强"，用户不满意。
正面案例: 用资金流数字、PE对比、均线排列数据逐一解释差异原因。

### ⚠️ mootdx 0.11.x bars() 参数名静默错误 (2026-07-07)
`client.bars(symbol=code, category=4, offset=40)` **不会报错但拿不到分钟数据**。
mootdx 0.11.x的bars()参数名从 `category` 改为 `frequency`，旧名被 `**kwargs` 静默吞掉，
默认退化为 `frequency=9`(日线)。日线请求碰巧结果一样，所以不易发现。

**正确写法**:
```python
bars = client.bars(symbol=code, frequency=9, offset=40)  # 9=日线
# frequency值表: 0=5min 1=15min 2=30min 3=60min 4=日线 5=周线 6=月线 8=1min 9=日线(默认)
```

**陷阱**: 如果你需要分钟K线(如1分钟/5分钟)，`category=8`会被吞掉退化为日线，
你会拿到日线数据但以为是分钟数据，量价分析全部失真。

**教训**: 所有 `client.bars()` 调用必须用 `frequency=` 不用 `category=`。
升级mootdx后如果发现K线数据异常(如分钟线全是日线)，先检查参数名。

### ETF分析与个股分析的差异(2026-07-03)

ETF(如159567港股创新药ETF)的分析与个股有以下不同:

1. **无PE/PB数据**: ETF本身没有PE/PB, 需要看底层成分股的估值水平
2. **无个股基本面**: 不需要分析公司业务/财报/概念, 但需要分析板块趋势
3. **无资金流数据**: 东财push2his不提供ETF资金流, 需要通过成交额+换手率判断
4. **重点不同**: 
   - 个股 → 量价+资金+估值+概念+消息
   - ETF → 板块趋势+技术面+大盘环境+行业轮动
5. **止损逻辑**: ETF波动通常小于个股, 可用均线止损(MA5/MA10)
6. **买入建议**: 用户会告知买入成本价, 需要给出明确的持有/减仓/止损方案

ETF分析简化流程:
  ① 获取K线+技术指标(MA/缩量比/R:R)
  ② 分析板块趋势(板块指数近期表现)
  ③ 判断大盘环境对板块的影响
  ④ 根据用户成本价给出具体操作建议(持有/止损/目标价)

### ⚠️ 止损位/支撑位/阻力位的有效突破必须以收盘价为准(2026-07-10核心教训)

**严重事故**: 7/9报告判定"止损位34.00已被有效跌破(今日最低33.60)"，但7/9收盘价实际是34.74(高于止损位34.00)。盘中最低33.60是假突破/探底回升，收盘站回=止损位未被有效跌破。7/10进一步确认: 最低33.33后强势拉回35.59(+2.10%)。

**规则**: 判断止损位/支撑位/阻力位是否被突破，**必须以收盘价为准**，不能以盘中最低/最高价为准。

| 判定标准 | 有效突破 | 假突破(盘中穿越但收盘站回) |
|----------|---------|--------------------------|
| 收盘价 | 在止损位/支撑位下方 | 在止损位/支撑位上方 |
| 操作 | 执行止损/确认突破 | 不执行止损，继续观察 |
| 后续确认 | 次日继续在下方=确认 | 次日拉回=假突破确认 |

**常见陷阱**:
- 盘中跌破止损位33.60 → 看到就判定"止损触发" → **错误!** 需看收盘
- 盘中突破阻力位 → 看到就判定"突破成功" → **错误!** 需收盘确认
- 收盘恰好在止损位(如34.00) → 模糊地带 → 偏向"未有效跌破"(宁可多观察一天)

**正确做法**:
1. 盘中跌破止损位 → **不立即判定止损触发**，等待收盘
2. 收盘价 < 止损位 → 止损有效触发，次日执行
3. 收盘价 > 止损位 → 止损未触发，继续持有/观察
4. 如需更严格: 收盘价连续2日 < 止损位 → 才确认有效跌破

**案例(泛微网络 7/9-7/10)**:
- 7/9盘中最低33.60 < 止损位34.00 → 报告判定"止损触发"
- 7/9收盘34.74 > 止损位34.00 → **实际未触发**
- 7/10收盘35.59(+2.10%) → 进一步确认止损未触发+出现缩量锤子线底部信号
- 如果7/9就执行止损，会错过7/10的反弹

### 缩量锤子线底部形态识别(2026-07-10新增)

**形态定义**: 在下跌趋势末端，出现一根**成交量极度萎缩**的**锤子线**(长下影+小实体+短上影)

**5项检查条件**:
1. 下跌趋势中(价格在MA20/MA30下方)
2. 成交量为近20-30日最低(或接近最低，<前5日均量的60%)
3. 下影线 > 实体 × 2(锤子线标准)
4. 上影线极短(< 实体的50%)
5. 收盘价接近当日最高价

**含义**: 卖盘枯竭(缩量) + 下方承接极强(长下影) + 多头尾盘占据优势(收盘=最高) = 经典底部形态

**强度判断**:
- 缩量程度: 量越低信号越强(近30日最低 > 近20日最低 > 近10日最低)
- 下影/上影比: 比值越大信号越强(>10倍=强, >50倍=极强)
- 收盘位置: 收盘=最高 > 收盘=开盘(阳锤子 > 阴锤子)

**⚠️ 必须次日确认**:
- 7/11收盘 ≥ 今日收盘且收阳线 → 底部确认，可小仓试探
- 7/11收盘 < 今日最低 → 锤子线失效，继续观望
- 7/11继续缩量且不破今日最低 → 底部信号增强

**案例(泛微网络7/10 — ⚠️后续发现数据错误!)**:
- 7/10报告记录: 成交量69,032手, 下影线2.26元, 上影线0.03元, 比值75倍
- **7/13实际核查: 成交量=201,801手(非69K!), 收盘33.83(-2.62%)(非+2.10%!)**
- 实际K线: O34.85 C33.83 H36.10 L33.33 → 上影线2.27 >> 下影线0.50
- **实际是"放量长上影阴线"(看跌), 不是锤子线(看涨)!**
- 结论: 报告数据错误导致K线形态判断180度反转
- **教训**: 见上方"子agent报告数据必须交叉验证"章节

### 子agent禁止编造K线数据,必须调用API验证(2026-07-15核心教训)

**严重事故**: 通富微电7/14报告中,子agent将6/25高点79.38错误归因为7/10开盘价79.39,

### 子agent报告数据必须交叉验证(2026-07-13核心教训)

**严重事故**: 7/10泛微网络报告中K线数据完全错误:
- 收盘价: 报告写35.59(+2.10%) → **实际33.83(-2.62%)** — 方向都反了!
- 成交量: 报告写69,032手 → **实际201,801手** — 3倍错误!
- K线形态: 报告定为"缩量锤子线"(看涨) → **实际是"放量长上影阴线"(看跌)**
- MA金叉: 报告说MA5>MA10 → **实际MA5远低于MA10，已死叉**
- 结论180度反转: 从"底部信号"变为"主力趁反弹出货"

**根本原因**: 子agent使用了错误的数据源或过时数据,导致整个分析结论错误。

**规则**: 子agent生成报告后,主agent在呈现给用户前,必须做以下交叉验证:

1. **收盘价方向验证**: 如果报告说"阳线(+X%)"但收盘价<开盘价,数据矛盾

3. **K线形态验证**: 
   - 锤子线: 收盘应在当日高位(下影线长+收盘≈最高)
   - 长上影阴线: 收盘应在当日低位(上影线长+收盘≈最低)
   - 如果报告的形态描述与价格位置矛盾,数据错误
4. **均线排列验证**: 如果报告说"MA5>MA10金叉"但实际MA5<MA10,数据矛盾

**快速验证脚本(主agent读取报告时用)**:
```python
# 从报告中提取关键数据,交叉验证
# 如果发现矛盾,必须重新获取数据再分析,不能直接呈现给用户
```

**影响范围**: 这是最高优先级的坑——错误数据导致错误结论,用户基于错误结论操作会造成实际亏损。

### ⚠️ 子agent会用理论价格fabricate K线数据(2026-07-15教训)

**严重事故**: 通富微电(002156)分析中,子agent报告7/10高点为79.39,但实际高点是77.67。
79.39是7/9涨停价72.17×1.1的理论涨停价,7/10实际从未触及。子agent用理论值代替了真实K线数据。

**案例(通富微电2026-07-15)**:

- ✅ 东财K线API验证: 7/10真实高点77.67
- 后果: "79.39突破"这个关键判断基于错误数据,实际应为"79.38突破(6/25高点)"

**规则**: 主agent在呈现子agent报告前,必须对关键价格点做以下验证:

1. **K线高低价验证**: 对报告中提到的"某日高点/低点",用东财K线API获取真实OHLC数据核实

2. **涨停价/跌停价不等于实际价格**: 涨停价=昨收×1.1(主板)/1.2(科创),是理论上限,实际未必触及
3. **如果报告中的关键价位无法通过API验证** → 标注"待验证"并用API核实后再呈现

### 盘中报告的数据时效性限制(2026-07-15教训)

**严重事故**: 通富微电(002156)7/14盘中报告基于11:30数据(70.00),建议"减仓/止损"。
但7/14下午发布H1业绩预增288%-337%,股价从68.40暴力拉至77.87(全天最高)。
报告的"减仓"建议被下午的反转完全否定。

**根本原因**: 盘中分析只能看到截至分析时点的数据,无法预见盘中突发催化(业绩公告/政策/资金异动)。

**规则**:
1. 盘中报告必须在结尾标注: "⚠️ 本报告基于盘中数据,收盘后可能有重大变化"
2. 如果分析时价格在日内低位(如跌5%+),必须考虑尾盘反弹的可能性
3. 如果分析时价格在日内高位(如涨5%+),必须考虑尾盘回落的可能性
4. 收盘价才是一天的最终定论——盘中极端价格(最高/最低)往往不代表收盘方向
5. 重大公告(业绩/重组/政策)通常在盘后发布,盘中分析无法预见

**正确做法**: 盘中分析给出操作区间(如"跌破77.87止损,站稳79.38加仓"),
而不是基于盘中瞬时价格的绝对判断(如"现在70.00,立即止损")。

### 超跌反弹权重被系统性低估(2026-07-21核心教训)

⚠️ 深科技/凯盛科技/通富微电复盘: 三只票全部在"极度看空"评级后次日涨停!

**问题**: 分析框架在个股暴跌30-40%+后,仍然给出极低评分(0.68/18.5/22分),
严重低估了超跌反弹的爆发力。

**根因**:
1. "主力出货完成=不会涨"逻辑有缺陷 → 新主力/游资可在低位重新建仓
2. 跌幅>30-40%时,技术性反弹概率被系统性低估
3. 板块系统性催化被忽略(今日3只票同时涨停→半导体/封测板块深V反弹)
4. PE估值框架在板块行情中暂时失效(高PE不是做空充分条件)

**正确判断框架**:
```
暴跌后评分检查清单:
  ① 累计跌幅>30%? → 自动进入"超跌区间",降低继续看空权重
  ② 多只同板块票同时涨停? → 排查板块级催化(政策/资金/情绪)
  ③ 主力出货完成度>80%? → 从"看空"调为"中性"(抛压释放=可能见底)
  ④ KDJ-J值<0或RSI<25? → 超卖信号权重提升
  ⑤ 评分震荡>30分(1周内)? → 禁止极端方向判断
```

**案例**:
  深科技: 从66.20跌至37.80(-42.9%),评分0.68分→次日涨停+9.99%
  凯盛科技: 从25.10跌至13.80(-45%),评分0.89分→次日涨停+9.99%
  通富微电: 3天跌25.6%后评分18.5分→次日涨停+10%

### 板块联动效应必须纳入分析(2026-07-21教训)

当多只同板块票同时涨停/跌停时,说明有**板块级系统性催化**,
个股分析必须结合板块维度:
1. 搜索当日板块异动原因(政策/资金/情绪)
2. 判断是个股问题还是板块问题
3. 板块暴跌后可能仅隔1天就反弹(通富微电: 7/20封测暴跌→7/21板块反弹14%)

### 评分震荡模式的正确处理(2026-07-21通富微电教训)

通富微电评分: 70→38.5→80.5→33.5→18.5(剧烈震荡)
→ 两次重大方向性错误(过度看多+过度看空)

**规则**: 评分震荡>30分(1周内)时:
  - 禁止给出极端方向性判断(80+看多或<20看空)
  - 应降低仓位建议,缩小止损区间
  - 标注"该股波动大,分析框架不稳定"
  - 等待方向明确后再做判断

### 利好兑现出货的硬规则(2026-07-21通富微电案例)

⚠️ 通富微电7/14: H1业绩预增288-337%发布→当天收盘=全天最高77.87
→ 7/15天量293万手+84.70=天量天价见顶→3天暴跌-25.6%

**硬规则(不可主观判断)**:
  利好发布当天收盘=全天最高 + 次日成交量创近期新高
  → 自动触发"利好兑现=见顶"标记
  → 量价维度自动扣至2/10以下
  → 不因利好消息提高评分

### 盘中数据的局限性(2026-07-14通富微电教训)

⚠️ 7/14盘中报告基于11:30数据(70.00),建议"减仓/止损"
→ 7/14下午H1业绩催化→收盘77.87(全天最高),建议被完全否定

**规则**:
1. 盘中报告必须标注"⚠️ 基于盘中数据,收盘后可能有重大变化"
2. 盘中分析给出操作区间,不做绝对判断
3. 重大公告通常在盘后发布,盘中无法预见

### 出货完成度>80%应中性偏多(2026-07-21教训)

当出货完成度评估>80%时(净流出≈净流入),资金面评分应从"看空"调为"中性":
  - 出货完成=抛压释放=底部可能已到
  - 不应继续给出"清仓"建议
  - 应转为"观望等待底部确认"
  - 如果同时出现超卖信号(KDJ<0/RSI<25),可小幅上调评分

### 暴跌后止损策略需动态调整(2026-07-21教训)

当个股已暴跌40%+时,止损策略需要重新评估:
  - 在低位继续追止损可能"割在地板上"(凯盛科技案例: 从17.84跌到13.80止损,随后涨停)
  - 暴跌后应切换为"观察反转信号"模式,而非继续追止损
  - 真正的止损应在暴跌初期(跌幅<15%时)执行
